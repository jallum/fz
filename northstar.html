Incremental Fact Compiler

The compiler is a world of stable entities and derived facts. Code indexing creates module and function identity. A single deduped agenda fires small rules. Type facts, reachability facts, return-demand facts, and executable-plan facts are co-derived in one root-scoped semantic fixpoint.

Facts change. Changed facts wake rules. Rules publish owned contributions. The compiler is done when the agenda is quiet, not when a phase says it has visited everything.

Goal, Signal, Strategy

Goal

Produce an answer for a root request while processing only facts reachable from that root. Definitions can exist without becoming semantic or backend work.

Signal

Telemetry shows code indexing, fact changes, agenda coalescing, semantic closure, materialization, ABI derivation, and emission. Tests assert what did not become reachable as much as what did.

Strategy

Build from tiny executable contracts. For each example, write the facts that should appear, the jobs that should coalesce, the facts that must remain absent, and the artifact boundary that must not rediscover semantics.

Principles

  1. Identity is stable. Referencing a module or MFA allocates an ID. Later definitions fill or replace the state behind that ID.
  2. Parsing indexes declarations. A code contribution can eagerly produce expanded function ASTs without lowering bodies or resolving internal calls.
  3. Facts are separate from entities. A function record says what code AST defines it. Root-scoped facts say how that function behaves in a particular semantic closure.
  4. Rules are the unit of work. There is no global type pass and no global planning pass. There are small rule instances keyed by the facts they can derive.
  5. Agenda entries coalesce. A rule key can be enqueued many times while pending, but it runs once. After it runs, later fact changes may enqueue it again.
  6. Derived outputs are owned. Each rule owns its contributions. When inputs change, the rule recomputes and replaces its old contributions, allowing facts to shrink after redefinition.
  7. Semantics close before artifacts. Materialization, ABI derivation, and codegen consume a closed semantic fact set. They cannot ask type questions or discover new callees.

World Stores

Stores own durable identity. Facts describe what is currently known about those identities. The stores are deliberately small; most compiler knowledge lives in fact tables. 

struct World {

Code And Function AST

Code parsing and top-level indexing are allowed to be eager for a code contribution. The output is identity plus already-expanded function AST owned by the definition, not an interned AST arena and not lowered IR. By the time a function AST is recorded, all relevant macro expansion for that function has already happened.

Dispatch planning is its own owned layer. A named helper that can be reified into PatternGuardExpr plus nested PatternDispatchPlan becomes a reusable dispatch fact; there is no separate defguard surface and no receive-only matcher path. 

enum CodeState {
    Added {
        code: CodeId,
        default_module: ModuleId,
    },
    Indexed {
        code: CodeId,
        modules: Vec<ModuleId>,
        functions: Vec<FunctionId>,
        macros: Vec<FunctionId>,
    },
    Failed,
}

enum FunctionDef {
    Code {
        visibility: Visibility,
        clauses: Vec<ClauseAst>,
        namespace: Namespace,
        code: CodeId,
    },
    Macro {
        visibility: Visibility,
        clauses: Vec<ClauseAst>,
        namespace: Namespace,
        code: CodeId,
    },
    Intrinsic {
        signature: Signature,
        op: IntrinsicOp,
    },
    Generated {
        owner: FunctionId,
        body: FunctionAst,
        namespace: Namespace,
    },
}

Runtime and built-in package libraries are not special code classes. They enter through the same code indexing and function definition path as user code. Some definitions may be intrinsic leaves, but the code that contributes them is ordinary.

AST handles are not part of the baseline design. Interning syntax has a cost: allocation, lookup, lifetime management, and indirection. Add an AST arena only for a measured reason, such as a macro-generated tree shared by many definitions. The default is direct ownership by FunctionDef.

Namespaces

Name resolution is an append-only namespace chain. A namespace head is an integer savepoint into that chain. Entering a scope saves the current head. Defining or importing a name pushes a binding and returns the new head. Leaving the scope restores the saved head. Lookup walks from the current head backward; the first matching binding wins. 

type Namespace = Option<BindingId>;

struct NamespaceStore {
    bindings: Vec<NamespaceBinding>,
    prelude_head: Namespace,
}

struct NamespaceBinding {
    prev: Namespace,
    name: Symbol,
    value: BindingValue,
    span: Span,
}

enum BindingValue {
    Module(ModuleId),
    Function(FunctionId),
    Macro(FunctionId),
    Type(TypeId),
    Value(LocalId),
}

fn push(ns: &mut NamespaceStore, head: Namespace, binding: NamespaceBinding)
    -> Namespace
{
    let id = BindingId(ns.bindings.len());
    ns.bindings.push(binding.with_prev(head));
    Some(id)
}

fn lookup(ns: &NamespaceStore, mut head: Namespace, name: Symbol, kind: BindingKind)
    -> Option<BindingValue>
{
    while let Some(id) = head {
        let binding = &ns.bindings[id];
        if binding.name == name && binding.value.kind() == kind {
            return Some(binding.value);
        }
        head = binding.prev;
    }
    None
}

The prelude is not a code phase or scheduler mode. It is just the namespace head saved after bootstrap definitions are installed: compiler forms, intrinsic leaves, default imports, and any package-provided names that should be visible by default. User, runtime, and built-in package code still enters through ordinary code indexing; visible names are namespace bindings.

Facts

A fact key names a table slot. A rule contributes a value to one or more fact slots. The aggregate value is the join of all current contributions for that key. Most slots have one owner; lattice slots such as type facts may have many contributing rules. 

struct FactSlot {
    aggregate: FactValue,
    fingerprint: Fingerprint,
    contributions: HashMap<JobKey, FactValue>,
}

struct FactChange {
    key: FactKey,
    old_fingerprint: Fingerprint,
    new_fingerprint: Fingerprint,
}

The Agenda

The agenda is one queue plus one set. The set prevents duplicate pending work. Popping a job removes it from the set, so a later fact change can enqueue it again. 

struct Agenda {
    queue: VecDeque<JobKey>,
    queued: HashSet<JobKey>,
}

fn enqueue(agenda: &mut Agenda, job: JobKey) {
    if agenda.queued.insert(job) {
        agenda.queue.push_back(job);
    }
}

fn pop(agenda: &mut Agenda) -> Option<JobKey> {
    let job = agenda.queue.pop_front()?;
    agenda.queued.remove(&job);
    Some(job)
}

Job Keys

Jobs name rule instances, not output facts. Several jobs may contribute to the same fact family, and one job may publish several facts. 

enum JobKey {
    IndexCode(CodeId),
    LowerFunction(FunctionId),

    SeedRoot(RootId),
    AnalyzeActivation(ActivationKey),
    AnalyzeCallSite(CallSiteKey),
    ResolveDirectCall(CallSiteKey),
    ResolveProtocolCall(CallSiteKey),
    DeriveReturnNeed(CallSiteKey),
    DeriveExecutable(CallSiteKey),
    PlanExecutable(ExecutableKey),
    CheckSemanticClosure(RootId),

    Materialize(RootId, SemanticEpoch),
    DeriveAbi(RootId, SemanticEpoch),
    Emit(RootId, Target, SemanticEpoch),
}

Subscriptions And Waiters

A running job records what it read and what it could not read yet. Reads become subscriptions. Missing required facts become waiters. When a fact changes or appears, only interested jobs are enqueued. 

struct DependencyIndex {
    reads: HashMap<JobKey, HashSet<FactKey>>,
    subscribers: HashMap<FactKey, HashSet<JobKey>>,
    waits: HashMap<JobKey, HashSet<FactPattern>>,
    waiters: HashMap<FactPattern, HashSet<JobKey>>,
    outputs: HashMap<JobKey, HashSet<FactKey>>,
}
while let Some(job) = agenda.pop() {
    let outcome = track_reads_and_waits(|| run_rule(world, job));

    if let Some(error) = outcome.fatal {
        telemetry.emit_diagnostic(error, job);
        return Fatal;
    }

    deps.replace_reads(job, outcome.reads);
    deps.replace_waits(job, outcome.waits);

    let changes = facts.replace_contributions(job, outcome.outputs);
    deps.replace_outputs(job, changes.output_keys);

    for change in changes.changed_slots {
        for subscriber in deps.subscribers(change.key) {
            agenda.enqueue(subscriber);
        }
        for waiter in deps.waiters_matching(change.key) {
            agenda.enqueue(waiter);
        }
        for root in roots_touched_by(change.key) {
            agenda.enqueue(CheckSemanticClosure(root));
        }
    }

    for root in outcome.completed_semantic_roots {
        agenda.enqueue(CheckSemanticClosure(root));
    }
}

Replacing contributions is what makes redefinition tractable. If a function body changes and a rule no longer derives an old call edge, that old contribution is removed. The aggregate fact changes, subscribers wake, and the blast radius propagates only as far as fingerprints change.

Errors are not facts. If any job returns a fatal error, the engine emits a diagnostic event through telemetry and stops the run. There is no separate diagnostics store to reconcile at closure time.

Rule Strata

Strata are write boundaries, not whole-program passes. The same agenda drives them all. Artifact rules are gated by SemanticClosed; semantic rules never wait for codegen.

Stratum Rules Publishes Cannot Do
Definition Index source, expand top-level forms, allocate placeholders, define modules/functions/macros/intrinsics. Code/module/function facts, namespace bindings, and owned expanded function AST. Lower every body or infer semantics.
Body Lower one demanded function body from its owned expanded AST and frozen namespace head. Lowered body, local value IDs, callsite IDs, generated function definitions. Resolve all possible callees or decide backend representation.
Dispatch Reify dispatch-pure helpers and ordered entry matching into shared dispatch plans. GuardDispatch(function) and EntryDispatch(function) facts, plus later body-site dispatch facts. Type bodies, plan executables, or emit code.
Semantic Analyze activations, callsites, protocol dispatch, return needs, executable specs, blocks, effects, and callable entries. Type facts and planning facts in one root-scoped fixpoint. Emit code or depend on artifact facts.
Artifact Materialize, derive ABI, emit interpreter/JIT/AOT artifacts. Executable artifacts keyed by root and semantic epoch. Discover new semantic reachability, callee selection, types, or effects.

Semantic Rules

This is the important collapse: type inference and demand planning are rule families in the same semantic stratum. Their outputs can feed each other until the agenda is quiet.

Rule Reads Publishes
SeedRoot RootEntry(root) Publishes the root entry fact and schedules the first closure walk once the entry definition exists.
AnalyzeActivation Activation, FunctionDefined, LoweredBody, EntryDispatch(function), and any nested guard-dispatch facts it references. Pattern reachability, value types, callsite instances, argument facts, return type contribution.
AnalyzeCallSite Callsite form, local value types, lexical env facts. Direct callee candidates, protocol receiver facts, callable-value facts, initial return needs.
ResolveDirectCall MFA reference, argument facts, function definition state. SelectedCallee and callee Activation.
ResolveProtocolCall Protocol receiver type, protocol impl registry, argument facts. Selected implementation, protocol telemetry, implementation activation.
DeriveReturnNeed Continuation shape: value use, destructure, branch test, tail return, backend root target. Return demand such as Value, TupleFields(n), BoolTest, or NoValue.
DeriveExecutable Selected callee, argument facts, return need. Executable(root, fn, input, need) and caller-to-callee executable edge.
PlanExecutable Executable, lowered body, value types, return needs, selected call edges. Reachable blocks, effects, return contracts, callable entries, extern marshal obligations.
CheckSemanticClosure Root entry, activation analyses, return facts, selected callees, return needs. Republished root-owned Activation / Executable facts and SemanticClosed(root, epoch) when the semantic frontier is stable.

Closure And Artifacts

SemanticClosed(root, epoch) is the only semantic-to-backend handoff. The epoch fingerprints the closed set and the base revisions it read. Artifact jobs are ordinary agenda jobs, but their inputs are closed facts, not source or type questions. 

SemanticClosed(root, epoch)
  -> MaterializedProgram(root, epoch)
  -> AbiReady(root, epoch)
  -> Emitted(root, target, epoch)

Materialization may clone, fold, inline safe known calls, fuse blocks, and prune dead edges. It must not publish semantic facts. If materialization needs a callee, return type, effect, callable entry, or extern marshal fact that is absent from the closed semantic set, the compiler reports an incomplete-semantic-plan diagnostic.

Redefinition

Redefinition is not special. A new code contribution changes definition facts. Jobs that read old fingerprints wake. They replace their owned outputs. The blast radius stops where aggregate fact fingerprints stop changing. 

old FunctionDefined(foo/0) fingerprint = A
new FunctionDefined(foo/0) fingerprint = B

changed(FunctionDefined(foo/0))
  -> wake subscribers that read foo/0's definition
  -> recompute their owned outputs
  -> wake downstream subscribers only if aggregates changed

If foo/0 was defined but never reachable from a root, redefining it wakes no semantic work for that root. If a reachable caller starts calling foo/0, the callsite rule derives the first activation and executable facts for foo/0.

Worked Example: quicksort + foo

Start with fixtures/quicksort/input.fz and add one extra uncalled function: 

fn foo(), do: 42

The stress question is whether foo/0 becomes visible without becoming reachable. It should be defined and available to future references, but it should not be lowered, semantically analyzed, materialized, or emitted for main/0.

01Indexing defines five functions

IndexCode(quicksort) parses the code contribution and indexes top-level function declarations. It publishes function definitions with owned expanded AST: 

FunctionDefined(append/2, clauses = Ast[...])
FunctionDefined(partition/4, clauses = Ast[...])
FunctionDefined(qsort/1, clauses = Ast[...])
FunctionDefined(main/0, clauses = Ast[...])
FunctionDefined(foo/0, clauses = Ast[...])

No body has been lowered yet. In particular, there is no LoweredBody(foo/0), no Activation(root, foo/0, []), and no Executable(root, foo/0, [], Value).

02The root seeds one semantic island

The host asks for JIT execution of main/0. That creates a root and enqueues artifact work. Artifact work blocks on semantic closure; it does not drive semantic discovery itself. 

RootEntry(root, main/0, input = [], target = jit)

SeedRoot(root) publishes:
  RootEntry(root)

CheckSemanticClosure(root) then owns the root frontier:
  Activation(root, main/0, [])
  Executable(root, main/0, [], Value)

These facts enqueue semantic jobs for main/0 only: 

LowerFunction(main/0)
AnalyzeActivation(root, main/0, [])
PlanExecutable(root, main/0, [], Value)

03Main pulls only its reachable references

LowerFunction(main/0) reads the owned expanded AST for main/0 and publishes a lowered body with callsites for: 

qsort/1
dbg/1 intrinsic
Process.heap_alloc_stats/0
map indexing
Kernel.+/2

The list literal gives the qsort/1 callsite a precise argument fact: 

ValueType(root, main, literal_list) = nonempty_list(int)
ArgTypes(root, callsite qsort) = [nonempty_list(int)]

AnalyzeActivation(main) publishes the direct call edge, and the next closure walk extends the rooted frontier: 

SelectedCallee(root, qsort callsite) = qsort/1
Activation(root, qsort/1, [nonempty_list(int)])
Executable(root, qsort/1, [nonempty_list(int)], Value)

foo/0 is still only a definition fact. No rule has read it.

04Recursive calls coalesce in the agenda

Lowering and analyzing qsort/1 discovers partition/4, append/2, and two recursive calls to qsort/1: 

append(qsort(lo), [p | qsort(hi)])

Both recursive calls may try to enqueue the same canonical activation job after their input shapes join to list(int). The agenda keeps one pending copy: 

enqueue(AnalyzeActivation(root, qsort/1, [list(int)]))  // inserted
enqueue(AnalyzeActivation(root, qsort/1, [list(int)]))  // coalesced
enqueue(AnalyzeActivation(root, qsort/1, [list(int)]))  // coalesced

After the job runs, a later change to an input fact can enqueue it again. Coalescing only suppresses duplicate pending work; it never suppresses changed work.

05Type and planning facts converge together

partition/4 is the recursive type stress point. Its input-list shape drives clause selection; its accumulator values should not create an unbounded product of precise list instances. The type contribution joins accumulator slots to the list family while preserving empty-list versus cons-list dispatch precision. 

ReturnType(root, partition/4, [list(int), int, list(int), list(int)])
  = tuple(list(int), list(int))

The destructure in qsort/1 is a planning stress point: 

{lo, hi} = partition(p, rest, [], [])

The continuation shape publishes: 

ReturnNeed(root, partition callsite) = TupleFields(2)
Executable(root, partition/4, [...], TupleFields(2))
ReturnContract(root, partition executable) = tuple fields delivered separately

These are semantic facts derived in the same agenda drain as the return type. Codegen will later consume them; it will not rediscover tuple-field delivery.

06Runtime references are pulled by callsites

Process.heap_alloc_stats/0, dbg/1, map access, and arithmetic operators enter only because main/0 has reachable callsites using them. Direct dbg(x) is an intrinsic effect returning x; it should not make a runtime Kernel.dbg/1 wrapper live unless the program takes it as a function value.

07Semantic closure excludes foo

The root closes when no root-scoped semantic jobs are queued, no required semantic waiters remain unresolved, and the semantic fingerprint is stable. 

SemanticClosed(root, epoch Q)

The closed semantic set includes: 

main/0
qsort/1
partition/4
append/2
Process.heap_alloc_stats/0
required intrinsics/operators

The closed semantic set does not include: 

foo/0

Expected absent facts: 

LoweredBody(foo/0)
Activation(root, foo/0, [])
Executable(root, foo/0, [], Value)
ReachableBlock(root, foo/0, ...)
MaterializedBody(root, foo/0, ...)
NativeSymbol(root, foo/0)

08Artifacts consume the closed set

Materialize(root, Q) consumes only facts reachable in SemanticClosed(root, Q). It may inline safe calls, fold known branches, fuse blocks, and prune erased edges. It must not pull foo/0, because no semantic fact mentions foo/0. 

MaterializedProgram(root, Q)
AbiReady(root, Q)
Emitted(root, jit, Q)

09Redefinition cases

If a REPL source redefines uncalled foo/0, only its definition fingerprint changes. No root semantic subscriber read that definition, so the quicksort root remains closed. 

changed(FunctionDefined(foo/0))
  -> no quicksort-root subscribers
  -> no semantic recomputation
  -> old Emitted(root, jit, Q) remains current

If a later REPL source redefines main/0 to also call foo(), the changed main/0 definition wakes LowerFunction(main/0). The new lowered body publishes a foo/0 callsite, and the next closure walk adds: 

SelectedCallee(root, foo callsite) = foo/0
Activation(root, foo/0, [])
Executable(root, foo/0, [], Value)
LoweredBody(foo/0)
ReturnType(root, foo/0, []) = int

Unchanged qsort/1, partition/4, and append/2 facts keep their fingerprints, so their downstream artifacts do not need to be rebuilt.

The more interesting variant replaces the old sort path with foo(): 

fn main() do
  x = foo()
  dbg(x)
end

In that case LowerFunction(main/0) replaces its old owned lowered-body contribution. The old qsort/1 callsite fact disappears and a new foo/0 callsite fact appears: 

- CallSite(root, main, qsort callsite)
- SelectedCallee(root, qsort callsite) = qsort/1
- Activation(root, qsort/1, [nonempty_list(int)])
- Executable(root, qsort/1, [nonempty_list(int)], Value)

+ CallSite(root, main, foo callsite)
+ SelectedCallee(root, foo callsite) = foo/0
+ Activation(root, foo/0, [])
+ Executable(root, foo/0, [], Value)

The old qsort/1 semantic island then retracts from this root: downstream partition/4 and append/2 facts lose their last root-scoped owners and disappear from the new semantic epoch. Cached root-independent lowered bodies may remain, but they are not part of SemanticClosed(root, next_epoch) and are not emitted for the new root artifact.

Worked Example: Enum.reduce

Source: fixtures/enum_list_allocations/input.fz. This example exercises runtime-library references, protocol dispatch, recursive list reduction, a generated reducer bridge, a user lambda, closure/callable facts, and allocation-sensitive emission. 

fn main() do
  xs = [1, 2, 3, 4, 5]

  dbg(Enum.count(xs))
  dbg(Enum.member?(xs, 3))
  dbg(Enum.reduce(xs, 0, fn (x, acc) -> acc + x end))

  stats = Process.heap_alloc_stats()
  dbg(stats)
end

01Main defines a generated reducer function

Source indexing defines main/0. Lowering main/0 creates a generated function for the reducer lambda, with a frozen namespace head and owned AST body: 

FunctionDefined(main/0)
FunctionDefined(lambda#reduce_sum/2, Generated { owner: main/0, body: FunctionAst, namespace })

The generated function is not analyzed merely because it exists. It becomes semantic work when the reachable Enum.reduce/3 path calls it through a callable value.

02Public Enum calls pull runtime code definitions

The lowered main/0 body has callsites for: 

Enum.count/1
Enum.member?/2
Enum.reduce/3
Process.heap_alloc_stats/0
dbg/1 intrinsic

Referencing Enum.reduce/3 allocates or finds its FunctionId. If the runtime source containing Enum has not been indexed yet, the resolver waits on that source's definition facts. When indexed, it publishes the requested function definitions with owned expanded AST. It does not lower unrelated runtime functions. 

FunctionDefined(Enum.reduce/3)
FunctionDefined(Enum.count/1)
FunctionDefined(Enum.member?/2)
FunctionDefined(Process.heap_alloc_stats/0)

Expected absent runtime facts at this point include Enum.sort/1, Enum.map/2, range protocol implementations, and map protocol implementations unless another reachable callsite asks for them.

03The list receiver selects the List implementation

The list literal gives the public reducer call precise argument facts: 

ArgTypes(root, Enum.reduce callsite):
  enumerable = nonempty_list(int)
  acc        = int
  reducer    = closure(lambda#reduce_sum/2)

Lowering and analyzing public Enum.reduce/3 discovers its wrapper shape: it calls Enumerable.reduce/3 with protocol-control state and a generated bridge reducer that turns a bare accumulator result into {:cont, next_acc}. 

FunctionDefined(lambda#enum_reduce_bridge/2, Generated { owner: Enum.reduce/3, ... })

Activation(root, Enumerable.reduce/3,
  [nonempty_list(int), tuple(:cont, int), closure(lambda#enum_reduce_bridge/2)])

ResolveProtocolCall reads the receiver fact nonempty_list(int) and publishes: 

ProtocolImpl(root, Enumerable.reduce/3, receiver = list(int)) = Enumerable.List.reduce/3
SelectedCallee(root, Enumerable.reduce callsite) = Enumerable.List.reduce/3
Activation(root, Enumerable.List.reduce/3, [...])

04Reducer recursion is a semantic SCC

The reachable path is: 

Enum.reduce/3
  -> Enumerable.reduce/3 protocol stub
  -> Enumerable.List.reduce/3
  -> List.reduce/3
  -> List.reduce_cont/3
  -> List.reduce_step/3
  -> lambda#enum_reduce_bridge/2
  -> lambda#reduce_sum/2
  -> Kernel.+/2

List.reduce_cont/3 and List.reduce_step/3 form a recursive SCC. The control-state type is an ordinary semantic fact: 

ControlState(int) =
  {:cont, int}
  | {:halt, int}
  | {:suspend, int}

The SCC is complete when its owned contributions stop changing. It must not force convergence by erasing to any; it must preserve enough tag precision for clause selection and final return projection.

05Callable and return facts are co-derived

The bridge reducer creates callable obligations. Those obligations are planning facts, but they are discovered while type facts are still flowing: 

CallableEntry(root, lambda#enum_reduce_bridge/2)
CallableEntry(root, lambda#reduce_sum/2)
Activation(root, lambda#reduce_sum/2, [int, int])
ReturnType(root, lambda#reduce_sum/2, [int, int]) = int
SelectedCallee(root, + callsite) = Kernel.+/2

The public reducer's continuation needs the unwrapped accumulator value: 

ReturnNeed(root, Enum.reduce callsite) = Value
ReturnContract(root, Enum.reduce/3 executable) = int

There is no handoff from a type fixpoint to a planner fixpoint. The callable facts, protocol edge, return type, return need, selected call edge, and executable spec all settle together in the same semantic agenda drain.

06Coalescing is precise, not fuzzy

Enum.count/1, Enum.member?/2, and Enum.reduce/3 may all use list enumeration machinery, but their reducer/callback shapes differ. The agenda coalesces identical job keys only; it does not merge merely similar function names. 

AnalyzeActivation(root, List.reduce_step/3, [list(int), count_state, count_reducer])
AnalyzeActivation(root, List.reduce_step/3, [list(int), member_state, member_reducer])
AnalyzeActivation(root, List.reduce_step/3, [list(int), sum_state, sum_reducer])

These are distinct if their input facts or callable facts differ. Repeated wakeups for the exact same key coalesce while pending.

07Semantic closure defines the artifact boundary

The closed semantic set includes the public Enum calls used by main/0, selected Enumerable.List callbacks, the small List reducer loop, reducer bridge, user lambda, Kernel.+/2, Process.heap_alloc_stats/0, and required intrinsics.

It excludes unrelated runtime functions and unrelated protocol implementations: 

Enum.sort/*
Enum.map/*
Enumerable.Range.*
Enumerable.Map.*
unused Kernel wrappers

Once SemanticClosed(root, epoch E) is published, artifact jobs can run. Materialization must not grow the reachable semantic set. Growth is a diagnostic.

08Allocation contract comes from facts

ABI derivation reads callable entries and closure captures from the semantic closure and materialized program. The expected native allocation story follows from those facts:

  • List literal: five cons cells for [1, 2, 3, 4, 5].
  • Public reduce wrapper: one wrapper closure for the bridge if the representation requires it.
  • List walk: no additional cons cells while reducing.
  • Stats snapshot: no unrelated map/range runtime paths pulled in before Process.heap_alloc_stats/0.

Telemetry Contract

Telemetry is always emitted. Event construction must be cheap: payloads carry facts, fingerprints, IDs, and already-owned lists. Handlers can count, summarize, sample, or ignore those values. The compiler should not compute telemetry-only aggregates.

fz.compiler.namespace.bound binding id, previous head, name, binding value, span Proves scope extension and default namespace construction without a prelude phase.
fz.compiler.fact.changed fact key, old fingerprint, new fingerprint, owner job Proves propagation is driven by changed facts.

Correctness Invariants

  • Stable keys: every entity, fact, and job key must be deterministic and independent of scheduling order.
  • No gratuitous interning: assign IDs to durable entities and derived facts, not to every syntax node. AST is owned directly unless sharing is proven valuable.
  • Owned outputs: a job rerun replaces exactly the contributions it owns; no stale derived fact survives by accident.
  • Fingerprint propagation: subscribers wake only when aggregate fact fingerprints change.
  • Fatal stops: any job error emits diagnostic telemetry and aborts the run; closure never masks errors.
  • No speculative specs: executable facts appear because semantic rules derive them from reachable callsites and return needs.
  • No backend discovery: artifact jobs cannot publish semantic fact families.
  • Uniform code: user code, runtime libraries, and built-in packages are code contributions, not scheduler special cases.
  • Prelude is a namespace: default visibility is a saved Namespace after bootstrap bindings, not a compiler phase.
  • Cheap telemetry: events carry already-owned values and lists; handlers compute summaries when needed.
  • Absent facts matter: tests should assert that uncalled functions and unrelated runtime implementations remain absent from semantic/artifact facts.

Open Stress Points

  • Retractions: define exact contribution replacement and aggregate recomputation semantics for facts with multiple owners.
  • Closure detection: make CheckSemanticClosure(root) precise in the presence of blocked waiters and roots sharing facts.
  • Protocol naming: choose stable implementation MFA names so protocol facts are debuggable and reproducible.
